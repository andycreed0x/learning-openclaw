Extendiendo las capacidades de tus agentes

Bienvenidos a la Sesión 5. En esta sesión exploraremos cómo las herramientas, los skills y el sandbox amplían las capacidades de los agentes de OpenClaw. Veremos cómo los agentes pueden interactuar con el mundo real de manera segura.

¿Qué son las Herramientas?

Las herramientas extienden lo que los agentes pueden hacer más allá de la conversación

Sin herramientas
El agente solo puede responder con texto basado en su conocimiento del modelo.
Con herramientas
El agente puede navegar la web, leer archivos, ejecutar código, enviar notificaciones y mucho más. 
 Sin herramientas:
 Usuario: "Lee mi email"
 Agente:  "No puedo hacer eso"

 Con herramientas:
 Usuario: "Lee mi email"
 Agente:  [usa herramienta IMAP]
 Agente:  "Tienes 3 emails nuevos..."
Las herramientas son el puente entre la inteligencia del agente y el mundo real. Sin herramientas, un agente solo puede generar texto. Con herramientas, puede interactuar con servicios externos, el sistema de archivos, la web y mucho más. Esto es lo que convierte a un chatbot en un verdadero asistente.

Categorías de Herramientas

Categoría Descripción Ejemplos
Navegador Navegación web y scraping Navegar, captura, extraer
Canvas Creación de contenido visual Dibujar, generar imágenes
Node Operaciones del sistema Leer/escribir archivos, HTTP
Cron Tareas programadas Recordatorios, chequeos periódicos
Webhooks Endpoints HTTP Recibir triggers externos
Exec Ejecución de código Ejecutar scripts, comandos
OpenClaw organiza sus herramientas en seis categorías principales. La herramienta de Navegador permite la navegación web automatizada. Canvas permite crear contenido visual. Node maneja operaciones del sistema como archivos y solicitudes HTTP. Cron gestiona tareas programadas. Webhooks recibe solicitudes externas. Y Exec ejecuta código y comandos del sistema.

Flujo de Ejecución de Herramientas

graph TB A["El agente recibe solicitud"] --> D{"¿Necesita herramienta?"} D -->|"No"| R["Respuesta directa"] D -->|"Sí"| T["Seleccionar herramienta"] T --> S{"¿Sandbox activado?"} S -->|"Sí"| SB["Ejecutar en sandbox"] S -->|"No"| EX["Ejecutar directamente"] SB --> RES["Resultado"] EX --> RES RES --> A2["Agente procesa resultado"] A2 --> RESP["Respuesta final"]
El agente decide autónomamente si necesita una herramienta y cuál usar, basado en la solicitud del usuario y las herramientas disponibles.
Este diagrama muestra el flujo completo de ejecución de herramientas. Cuando el agente recibe una solicitud, primero decide si necesita una herramienta. Si no la necesita, responde directamente. Si la necesita, selecciona la herramienta apropiada. Luego, dependiendo de la configuración del sandbox, la herramienta se ejecuta en un entorno aislado o directamente. El resultado vuelve al agente para componer la respuesta final.

Herramienta Navegador en Detalle

Chromium headless vía Puppeteer

  • Navegar a cualquier URL
  • Capturar pantalla de páginas completas
  • Hacer clic en elementos específicos
  • Rellenar formularios automáticamente
  • Extraer contenido estructurado
  • Esperar elementos dinámicos
// Ejemplo de uso por el agente
{
  tool: "browser",
  action: "navigate",
  url: "https://ejemplo.com",
  // El agente puede encadenar acciones:
  // navegar -> extraer -> capturar
}
Nota: El navegador se ejecuta en modo headless. No se abre una ventana visible — todo sucede en segundo plano.
La herramienta de navegador es una de las más potentes. Utiliza Puppeteer para controlar un navegador Chromium headless. Puede navegar a cualquier URL, tomar capturas de pantalla, interactuar con elementos de la página como clics y formularios, y extraer contenido. Todo esto ocurre en segundo plano sin interfaz visual, lo que lo hace perfecto para automatización.

Herramienta Exec

Ejecutar comandos del sistema de forma controlada

Capacidades
  • Ejecutar comandos del sistema
  • Comandos permitidos configurables
  • Aislamiento sandbox disponible
  • Timeout configurable
Precaución
Siempre usa sandbox en producción. Limita los comandos permitidos al mínimo necesario. 
// openclaw.json5
{
  tools: {
    exec: {
      enabled: true,
      // Lista blanca de comandos
      allowedCommands: [
        "ls", "cat", "grep",
        "curl", "node", "python3"
      ],
      // Timeout en milisegundos
      timeout: 30000,
      // Forzar sandbox
      sandbox: true
    }
  }
}
La herramienta Exec permite al agente ejecutar comandos del sistema operativo. Es muy potente pero también potencialmente peligrosa, por eso es crucial configurarla correctamente. Puedes definir una lista blanca de comandos permitidos, establecer un timeout y forzar la ejecución en sandbox. En producción, siempre deberías limitar los comandos al mínimo necesario y activar el sandbox.

¿Qué son los Skills?

Paquetes reutilizables de prompt + herramienta

Piensa en ellos como "apps"
Cada skill combina instrucciones (prompt) con capacidades (herramientas) para resolver una tarea específica de forma empaquetada y reutilizable.
Ejemplo: Skill "resumen"
Prompt: "Resume el contenido de la URL proporcionada"
Herramienta: Navegador (para leer la página)
Resultado: Resumen estructurado del contenido 
// Ejemplo de skill personalizado
// /workspace/skills/traducir.json5
{
  name: "traducir",
  description: "Traduce texto entre idiomas",
  prompt: `Eres un traductor profesional.
    Traduce el texto proporcionado
    al idioma solicitado.
    Mant√©n el tono y estilo original.`,
  tools: ["exec"],
  // Activar por palabra clave
  trigger: "/traducir"
}
Los skills son uno de los conceptos más potentes de OpenClaw. Piensa en ellos como aplicaciones para tu agente. Cada skill empaqueta un prompt específico con las herramientas necesarias para resolver una tarea concreta. Por ejemplo, el skill de resumen combina un prompt de instrucciones con la herramienta de navegador para leer y resumir páginas web. Puedes crear tus propios skills fácilmente.

Incluidos: Vienen con OpenClaw. Listos para usar desde el primer momento.
Workspace: Crea tus propios skills en la carpeta /workspace/skills/.
ClawHub: Repositorio comunitario donde puedes explorar, compartir e instalar skills creados por otros usuarios.
Los skills pueden venir de tres fuentes. Los skills incluidos o built-in vienen con OpenClaw y están listos para usar, como resumen, traducir y buscar-web. Los skills de workspace son los que tú creas en tu carpeta de workspace. Y ClawHub es el repositorio comunitario donde puedes encontrar e instalar skills compartidos por otros usuarios. Es como un marketplace de habilidades para tu agente.

Modos de Sandbox

Controla el nivel de aislamiento de la ejecución

Modo Descripción Seguridad Uso recomendado
off Sin sandbox Baja Solo desarrollo local
non-main Sandbox para agentes no principales Media Uso personal controlado
all Sandbox para todo Alta Producción y multi-usuario
Recomendación: Usa el modo all siempre que sea posible. El modo off solo debe usarse durante el desarrollo, nunca en producción.
El sandbox tiene tres modos de operación. El modo off desactiva el sandbox por completo, lo que significa que las herramientas se ejecutan con acceso completo al sistema. Esto solo es apropiado para desarrollo local. El modo non-main aplica sandbox solo a los agentes que no son el principal, útil cuando confias en tu agente principal pero no en los secundarios. El modo all aplica sandbox a toda ejecución, y es lo recomendado para producción y entornos multi-usuario.

Arquitectura del Sandbox

Aislamiento mediante isolated-vm

graph TB subgraph "Proceso Principal" AG["Agente"] end subgraph "Sandbox (isolated-vm)" EX["Ejecución de Código"] FS["Acceso FS Limitado"] NET["Restricciones de Red"] end AG -->|"llamada aislada"| EX EX -->|"resultado"| AG
Dentro del sandbox
  • Memoria limitada
  • CPU con timeout
  • FS restringido a workspace
  • Red filtrada
Fuera del sandbox
  • Acceso completo a memoria
  • Sin límites de CPU
  • FS completo
  • Red sin restricciones
La arquitectura del sandbox utiliza isolated-vm para crear un entorno de ejecución completamente aislado. Dentro del sandbox, el código tiene memoria limitada, un timeout de CPU, acceso al sistema de archivos restringido solo al workspace, y conexiones de red filtradas. Esto significa que incluso si un agente intenta ejecutar código malicioso o una inyección de prompt logra manipular al agente, el daño está contenido dentro del sandbox.

Conclusiones Clave

Las herramientas dan capacidades del mundo real
Navegador, Exec, Cron, Webhooks y más — tus agentes pueden interactuar con servicios externos, el sistema de archivos y la web.
Los Skills son paquetes reutilizables
Combinan prompts y herramientas en unidades empaquetadas. Crea los tuyos o instala desde ClawHub.
El Sandbox protege tu sistema
Aislamiento con isolated-vm. Tres niveles de seguridad. Siempre usa all en producción. 
  Herramientas + Skills + Sandbox
  = Agentes potentes Y seguros
Recapitulemos los puntos clave de esta sesión. Las herramientas transforman a los agentes de simples chatbots a asistentes capaces de interactuar con el mundo real. Los skills empaquetan esas capacidades de forma reutilizable. Y el sandbox asegura que todo esto ocurra de forma segura. La combinación de estos tres elementos es lo que hace de OpenClaw una plataforma potente y segura.
